# 09 · 装好后 5 分钟验收 SOP(必跑)

> ⚠️ **不跑这个 SOP 不要批量发邮件。** 没验证过的 skill 输出 + 没替换的占位符 = 客户收到 `Hi {联系人:名称}` = 当场失信。

## 4 关 · 每关必过

### 关 1 · 触发(1 分钟)

在 Claude Code 项目里说:

> "我要用来发信开发北美亚洲食品分销商,我做手工面,先帮我找客户类型词。"

**预期**:Claude 应自动加载本 skill 的 `SKILL.md` + 路由到 `references/01-customer-search.md`,输出客户类型词池 + 中英对照(**只走"搜客户"阶段** · 不会一并要标杆 / 公司资料 — 那些是写邮件阶段才要)。

**没触发?**
- 检查 `laifaxin-outreach/` 是否在 `.claude/skills/` 或 `~/.claude/skills/` 下
- 检查 `SKILL.md` frontmatter 的 `name` 是否为 `laifaxin-outreach`
- 重启 Claude Code

### 关 2 · 变量真实性(2 分钟)· **最容易翻车**

skill 出的邮件里 `{联系人:名称}` / `{First Name}` 是 **占位示例**,**不是来发信的真实变量**。

录入流程必须是:

1. 在来发信 → 设置 → **邮件模板** → 进对应目录 → 新建模板
2. 把 skill 出的 HTML **粘贴到内容编辑器**(此时正文里 `{联系人:名称}` 是普通文字)
3. **删掉**那段 `{联系人:名称}` 占位符
4. 在原位置点编辑器顶部 **【插入变量】** 按钮 → 下拉里选真实字段(如 `名字` / `姓氏` / `公司名`)
5. 系统自动插入真变量(显示为带颜色的高亮 chip,不是普通文字)

出处:[laifa.xin/zhinan/email-templates](https://www.laifa.xin/zhinan/email-templates) L62-63 "**🔴 变量名打错(如 `{姓氏}` 写成 `{姓}`)→ 变量不替换,客户收到 `Dear {姓}` 完蛋 → 必用【变量】按钮插入,别手打,防拼写错**"

**自检**:模板预览页里**变量位置是高亮 chip,不是 `{XX}` 字面字符串** → 通过。

### 关 3 · 预览(1 分钟)

来发信邮件编辑器顶部点 **【预览】**:

- ✅ HTML 钩子(黄底红字 span)正常渲染?
- ✅ Bullet 列表有缩进 + 圆点?
- ✅ 段落之间有空隙?(呼吸感 · 见 `references/04 § 1.2`)
- ✅ Emoji 显示正常(标题里的 🍜 / 📊 等)?
- ✅ 没有 raw HTML 标签外露(如 `<p>` 显示成文字)?

任一不通过 → 编辑器 HTML 源码模式排查 + 修。

### 关 4 · 自发测试(1 分钟)

来发信邮件发送界面:**至少给自己邮箱发一封测试**(出处:[email-templates L68](https://www.laifa.xin/zhinan/email-templates) 官方建议)。

> **最低门槛**:1 个主用邮箱看变量替换 + 渲染都对。
> **推荐**:再加 1 个非 Gmail 客户端(如有 — 公司用 Outlook / 个人用 Apple Mail)交叉验证 fallback 效果。
> 三端全齐**不是硬要求**,根据手边账户量力而行。

**收到后查**:

| 客户端 | 看什么 |
|---|---|
| Gmail Webmail | 黄底红字钩子是否渲染 · 变量是否替换为真名 · emoji 是否正常 |
| Outlook.com / Outlook 桌面版 | 同上 · Outlook 桌面 Word 渲染内核 CSS 支持有限,部分样式可能被改写或忽略 → 因此 `references/04 § 3` 强制 **A/B 双模板默认挂载**(A=`<span>` 黄底红字 / B=`<b>` 加粗 fallback)· 来发信"步骤"内两版同时挂,系统随机发 · 这里看的就是 B 模板是否正常 |
| Apple Mail | 通常支持 inline style · 建议自测 |
| 手机端(iOS Mail / Gmail App) | 行宽是否友好 · 字体可读 · 单 CTA 是否清晰 |

**出现问题**:
- 变量未替换:回关 2,重做【插入变量】
- 钩子未渲染:回 `references/04 § 3 客户端兼容性矩阵` 看 fallback 表 · 多挂一版 `<b>` 模板
- 进了垃圾箱:检查 `SKILL.md § A.2` 签名是否过载 / 是否含外链 / 是否含触发反垃圾的词(`free` / `guarantee` / 全大写主题)

## 通过后 · 才能批量发

4 关全 ✅ → 才能在来发信"智能跟进计划"里:

1. 把模板挂进步骤
2. 按 `references/07 § 6` 加客户 + 激活计划(右上角开关变蓝)

> 出处依据:[laifa.xin/zhinan/email-templates L68](https://www.laifa.xin/zhinan/email-templates) "🟡 模板未测试就群发 → 变量错误 / 格式崩 = 1000 客户一起翻车 → 群发前**给自己发一封测试**,确认变量替换正确再批量发"

## 反面教材

- ❌ skill 装好直接 PR / 截图发朋友圈炫耀 — 没跑关 2/3/4 = 营销自残
- ❌ 关 1 触发不了就改 SKILL.md description 凑词 — 应先看 Claude Code 版本 + skill 目录权限
- ❌ 关 2 跳过 → "{联系人:名称}" 字面字符串发给 1000 客户 → 1000 个潜在客户当场拉黑
- ❌ 关 4 只在 Gmail 测试 → Outlook 客户全部钩子崩坏没发现 → 半个月后才发现回复率诡异低
