# 06 · 回复推进 playbook

> 客户回信了 → **不再是写开发信的事**。本 reference 从首封后开始,处理"对方回复 → 推进到样品 / 报价 / 试单"的标准动作。
>
> 来源:`2.邮件教程.jpeg` 真实运营案例 + 行业经验固化。

## 核心心法

1. **首封不硬卖,像买家一样去问**(已在 `05-first-round-rules.md` 体现)
2. **借对方回复确认 3 件事**:
   - 客户是不是真客户(业务对路)
   - 邮箱有效(没退信 + 真人回)
   - 负责人是不是这位(还是要转介)
3. **每一回复都是**"信息不对称的纠正机会" — 顺着对方话头多问一点

## 回复场景分类 + 应对脚本

### 场景 1 · 非负责人转介绍

**信号**:对方说"我不负责采购,请联系 X / 邮箱 Y / 转给 Z"

**动作**:
1. **立刻**给原回复人**短谢**(1-2 句,不卖)
2. **新开 thread 给新联系人**,引用上一位的转介("X kindly referred me to you")
3. 新邮件**仍按首轮规则**(≤ 100 词 / 1 CTA)

**英文模板**:
```
Hi [新负责人名],

[原联系人] kindly suggested I reach out to you regarding [话题].

I'm [我名] from [我司], a [背书一句] supplier of [品类].

Would it be helpful if I sent over our catalog tailored to [他们公司类型]?

Best,
[我名]
```

### 场景 2 · 给了竞品型号 / 现用产品

**信号**:"We currently use [品牌 X] / [型号 Y]"

**动作**:
1. **不要立刻报价**(没拿到他们真痛点之前)
2. **再问一层细节**:
   - 当前哪些方面满意?
   - 哪些方面想改善?
   - 用量大概?
   - 上次换供应商是什么时候?
3. 拿到信息后再发**对标数据**(策略 4 对标竞争)

**避坑**:`We are cheaper than [品牌 X]` — 一句话毁信任,显得 desperate

### 场景 3 · 索要样品 + 提到样品费/运费异议

**信号**:"Can you send free samples? Who pays shipping?"

**动作**(三选一,按客户体量):
1. **大客户**:免样品 + 我方付运费,但要求收件后**填一份 5 题问卷**(锁信息)
2. **中等**:样品免费,运费对付(到付方式给我方账号)
3. **小客户**:把"送样"切换成"**首单 MOQ 减半试单**" — 客户付钱,但门槛低 · 适合**起订量门槛高**的品类(化工 / OEM / 定制件)

**英文模板(场景 1 大客户)**:
```
Happy to send samples.

To make sure we send the right SKUs, could you share:
- Your annual purchase volume estimate
- Top 3 products you're currently sourcing
- Your preferred packaging size

We cover the sample cost; shipping is to your forwarder.

Best,
[名]
```

### 场景 4 · 要价格 / 要报价表

**信号**:"Please send pricing" / "Send your price list"

**动作**:
1. **不要直接发完整 price list**(失去后续推进锚点)
2. **反问 2 个 qualifier**:
   - 主要采购 SKU 是哪几个?
   - 年采购量大概多少?
3. 拿到信息后,**只报这几个 SKU 的分级价**(MOQ 1 / 5 / 10 柜),每柜价不同 → 引导他们 commit 数量

### 场景 5 · 要"先做 OEM / 定制"

**信号**:"We need our own brand / private label / custom recipe"

**动作**:
1. 立刻确认**最低起订量**(我方底线)
2. 提供**3 阶段路径**(按"改动深度"由浅到深):
   - **Stage 1 · 现成成熟款**(走货快,只换 label / 装箱方式):
     - 食品 → 现有库存 SKU
     - CNC → 已批量生产的标准件
     - LED → 现成在售型号
     - 化工 → 现成在售配方
   - **Stage 2 · 低改动定制**(不重开模 / 不重做配方):
     - 食品 → 贴牌 + 改包装 + 改 logo
     - CNC → 基于现有图纸 / 工艺的小改版(尺寸 / 表面处理微调)
     - LED → 改外壳颜色 / 接线方式 / 包装
     - 化工 → 改 pack size / 改标签 / 改规格分级
   - **Stage 3 · 高改动定制**(慢 + MOQ 高 + 需开模费 / 打样费 / 配方研发):
     - 食品 → 定制配方
     - CNC → 全新开模 / 定制结构
     - LED → 全新光学 / 结构设计
     - 化工 → 配方研发 / 安全数据表重做
3. 建议**从 Stage 1 / 2 切入**,Stage 3 后置 — 客户首单先验证我方履约能力,再谈深度定制

### 场景 6 · 算年度账(对成本敏感客户)

**信号**:对方多次提价格 / 比较

**动作**:
1. 不卷单价
2. **算年度节省账**:
   - 我方价 - 当前供应商价 = X / 单位
   - 年采购量 = Y 单位
   - 年节省 = X × Y = $Z
3. 把 $Z 写在邮件首屏(用 `<b><u>$Z annual savings</u></b>`)
4. 配一个"试 1 柜算账验证"的低门槛 CTA

## 通用结构约束

无论哪个场景:

- 回复仍**保持 ≤ 150 词**(可以比首封略长,但别失控)
- **永远只给 1 个 CTA**(最多 2,前提是 yes/no 二选一)
- 签名规则**与首封一致**(只英文名)
- 中文备注**只给操作者看**,不发出去

## 回复后 · 来发信端的接管 / 暂停 / 恢复(handoff)

> 写完回复稿不是结束。客户回信后,该联系人在**智能跟进计划**里的后续 step 怎么处理? 必须配套决策。

| 客户状态 | 来发信端动作 | 出处 |
|---|---|---|
| 客户首次正向回复(询价 / 索样 / 约通话) | 该联系人**手动改"已成交"或自定义"询盘中"标签** → 配合"未发送触发器"(`07 § 5`)排除该标签 → 后续 step 不再触达 | [docs/zhinan/email-sequence-guide L297-302](https://www.laifa.xin/zhinan/email-sequence-guide) |
| 客户中性回复(不是负责人 / 不需要) | 视情况:转介给新人 → 新人**重新加入**计划走首轮;非负责人 → 该联系人加"已转介"标签,触发器排除 | 同上 |
| 客户拒绝(已有供应商 / 不做这品类) | 该联系人加"已拒"或"黑名单"标签,后续 step 永久排除 | 同上 |
| 客户 OOO 自动回复 | **不算回复 · 不触发任何动作**;Tony 必须**确认"已完成触发器(回信即停)"是关闭的**(`07 § 5`/SKILL.md A.2),否则计划会被 OOO 误停 | 同上 |
| 多线程同公司 | 已配置"公司触发器 = 标记其他联系人为未发送"(`07 § 5`)→ 客户一人回复,同公司其他人 step 自动停 | 同上 |

### 何时整体暂停计划

1. **总退信率 > 10%** → 暂停 + 触发 AI 评分**全面扫描**重筛(参 `08 § interrupt`)
2. **总退订率 > 0.5%** → 暂停 + 改模板(反垃圾特征)
3. **OOO 占比 > 5%** → 客户在度假 / 节假日 → 暂停 1-2 周再继续

### 何时恢复

- 上一步的根因解决后(换通道 / 改模板 / 度假结束)→ 来发信"列表页"找到该计划 → 右上角开关重新激活
- **客户已转人工后**不要让该联系人重新进序列(会和人工跟进打架)

## skill 行为约定

1. 用户提供客户回复原文时,**先打分类**(场景 1-6 哪一个 + 是否混合)
2. 混合场景拆解后**逐场景出策略**,不要写一封"什么都说"的回复
3. **下一步目标必须显式写出**(拿邮箱 / 拿用量 / 排样品 / 上报价 / 上试单 / 拿决策人)
4. 输出格式:
   ```yaml
   classification: scenario_3 (sample objection)
   next_goal: "锁年采购量 + 出对路样品清单"
   english_reply: |
     ...
   chinese_notes: |
     ...(策略意图 + 风险提示)
   risks:
     - "客户可能找借口要更多样品"
     - "对策: 锁定 SKU 数 ≤ 5"
   ```

## 反面教材

❌ 收到"非负责人转介" → 继续给原联系人发第二轮 — **浪费机会**,直接换人
❌ 收到"客户用竞品" → 一封长信对比 10 个维度 — 客户没空看,直接问 2 个 qualifier
❌ 收到"要 free samples" → 直接同意 + 走 DHL 国际件 — 成本失控,锁信息再说
❌ 收到"要 price list" → 直接发 30 页 PDF — 没了后续 thread
