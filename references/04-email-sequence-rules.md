# 04 · 邮件写作 SOP · 长提示词拆解版

> 本文是原始长提示词(`3.提示词.txt`)的**结构化拆解**。
> skill 在生成邮件时,把它作为"**默认写作策略**"参考(不是来发信产品规则)— 可被首轮规则覆盖(参 `05`),也可被用户偏好覆盖。

## § 0 · 优先级声明

| 规则源 | 优先级 |
|---|---|
| **首轮特例**(`references/05-first-round-rules.md`) | 最高(覆盖本文 § 1.2 的 bullet 强制 / § 1.2 的 5 步流死板)|
| **本文默认值**(§ 1-7) | 默认(无首轮特例 / 无用户偏好时按这套)|
| **用户偏好**(如"我们行业客户更接受长邮件 / 不接受 emoji")| 可覆盖本文 § 1.3 极简 / § 1.2 视觉 |

## § 1 · 核心创作准则

### 1.1 深度共情

- 视角锁定:**第一思考点永远是"收件人最关心什么、最头疼什么"**
- 忘掉"我要推销什么",聚焦"我能帮你解决什么"
- 开场白必须直击一个**具体、真实**的业务痛点(供应链延迟 / 成本飙升 / 环保法规 / 创新瓶颈 ...)

### 1.2 呼吸感公式(扫视友好 · 强制框架)

#### ① 段落戒律

**一段一句**。每个段落**绝对不能超过一句话**。

#### ② 核心价值展示

**第 2 轮及之后默认**用项目符号 `<ul><li>...</li></ul>` · **2-3 个最佳**。
**首轮可极简**,不强制 bullet(参 `05-first-round-rules.md`)— 真实有效首轮往往只有 2-3 句话 + 1 个 CTA,加 bullet 反而拉长篇幅。

#### ③ 邮件流模板(5 步主体)

```
1. 开场痛点 (Opener - Pain Point)        — 1 句
2. 解决方案桥梁 (Bridge - Solution Intro) — 1 句
3. 核心价值列表 (Core Value - Bullets)    — 2-3 个 bullet
4. 引导式提问 (Closing Question)          — 1 句
5. 行动号召钩子 (The Hook)                — 1 句
```

#### ④ 视觉焦点层级(4 铁律)

| 层级 | 用法 | 标签 |
|---|---|---|
| 第一层 · 核心冲击点 | bullet 里最击中痛点 / 最强利益点 | `<b><u>...</u></b>` 或 `<b>...</b>` |
| 第二层 · 支撑关键词 | 产品名 / 认证 / 关键特性 | `<i>...</i>` 或 `<b>...</b>` |
| **数量铁律** | 全文(不含 CTA 钩子)被强调词 **总数 ≤ 4 个** | — |

案例:`A single point of contact that <b><u>slashes lead times</u></b>.` · `<i>ISO 9001 certified</i> CNC machining...`

### 1.3 极简主义

| 规则 | 值 |
|---|---|
| 字数 | **若用户未指定,首封 ≤ 100 个英文单词**(含签名)|
| 称呼 | **绝对禁止 `Dear`** · 用 `Hi {联系人:名称}`(⚠️ `{联系人:名称}` 是占位示例 · 录入来发信时**必用编辑器顶部【插入变量】按钮**替换 · 详见 `references/09-validation.md` 关 2) / `Hello` / 直接强开场 |
| 签名 | 只留**一个友好英文名**(如 `Alex`) · **严禁** 公司 / 职位 / 电话 / 链接 / banner image |
| 标题 | 5-7 个英文词 · 引发好奇或暗示解决方案 · 开头或结尾**单个**相关 Emoji |
| 语言 | 摒弃空洞商业术语 · 每个词像手术刀一样精准 |

签名极简的根因:**零营销感最大化规避垃圾邮件过滤器**。

## § 2 · 9 个策略编号(Strategic Lenses)

每一轮选 1 个策略作为该轮主基调:

| # | 策略 | 钩子类型偏好 | 用例 |
|---|---|---|---|
| 1 | 价值主张型 | Resource | 普适好处 + 产品核心 |
| 2 | 降本增效型 | Insight | 节约金钱 / 时间 / 效率 |
| 3 | 稀缺资源型 | Resource | 独家技术 / 特殊材质 |
| 4 | 对标竞争型 | Insight | 相对竞品的决定性优势 |
| 5 | 降低门槛型 | Resource | 低 MOQ / 快速打样 / 免费设计 |
| 6 | 趋势引领型 | Resource | 行业新趋势切入 |
| 7 | 环保合规型 | Qualification | 可持续 / 环保认证 / CSR |
| 8 | 交期优势型 | Qualification | 快速交货 / 稳定供应 |
| 9 | 服务保障型 | Resource | 售后 / 质保 / 技术支持 |

### 策略递进示范(8 轮)

```
痛点破冰 → 价值展示 → 建立信任 → 差异化
↓
轮 1 策略 8 (交期) → 轮 2 策略 2 (降本) → 轮 3 策略 9 (服务) → 轮 4 策略 4 (对标)
→ 轮 5 策略 7 (合规) → 轮 6 策略 6 (趋势) → 轮 7 策略 5 (门槛) → 轮 8 策略 1 (价值总结)
```

## § 3 · 钩子设计 · 3 原型(必选其一)

CTA 不是请求,是**下意识反应测试**。极低门槛,几乎不用思考。

### ① 资源型钩子(Resource Hook)

- **心理学**:互惠原则
- **适用**:策略 6 / 3 / 1 / 5 / 9
- **案例**:`...reply with Catalog to receive our latest exclusive designs.`

### ② 洞察型钩子(Insight Hook)

- **心理学**:信息差 + 好奇心
- **适用**:策略 2 / 4
- **案例**:`Want to see the data on how we saved [SimilarCo] 20%? Reply with Data.`

### ③ 资格型钩子(Qualification Hook)

- **心理学**:承诺与一致性
- **适用**:策略 7 / 8
- **案例**:`If your products arrived on time before Christmas, would you be interested? A simple Yes will do.`

### 视觉强调(默认 HTML 语法 + 兼容性 fallback)

钩子核心触发词**默认**用以下结构:

```html
... reply with <span style='background-color:yellow; color:red; padding:2px 4px; font-weight:bold;'>KEYWORD</span> to receive...
```

#### 客户端兼容性矩阵(基于事实,不可凭空断言)

| 客户端 | 渲染状态 | 出处 / 备注 |
|---|---|---|
| Gmail Webmail | ✅ 已验证可渲染(`background-color` / `color` / `font-weight` / `padding` 都受支持) | [Gmail CSS support](https://developers.google.com/workspace/gmail/design/css) |
| Apple Mail | 🟡 通常支持 inline style · 建议跨客户端自测 | 无官方矩阵 |
| Outlook.com (web) | 🟡 部分 inline style 受支持 · 建议自测 | 无统一官方矩阵 |
| Outlook desktop (Word render) | 🔴 已知会 strip / 改写部分 CSS · 建议 fallback | 业界常识 |
| 部分企业 SMTP / 安全网关 | 🔴 可能 strip 内联样式 | 业界常识 |

#### Fallback(降级到纯文本 / 半 HTML)

不确定客户端兼容性时,降级为:

```html
... reply with <b>CATALOG</b> to receive our latest exclusive designs.
```

或更保守:

```
... reply with CATALOG to receive our latest exclusive designs.
```

### 输出约束 · A/B 双模板(本 skill 强制做法)

**默认输出 A/B 双模板**,而不是只出一版:

- **模板 A · `<span>` 黄底红字版** — Gmail / 大多数 Webmail 表现最强
- **模板 B · `<b>` 加粗 fallback 版** — Outlook 桌面 / 保守企业邮箱兜底

在来发信"智能跟进计划 → 步骤"内**两个模板同时挂载**,系统随机用 → 自然分流 + 兼容性兜底。

**模板自检字段**(参 `assets/first-round-template.md` § 自检表)新增:
```yaml
hook_html_present: true       # A 模板 span 黄底红字
hook_fallback_present: true   # B 模板 <b> 加粗
both_templates_attached: true # 来发信步骤内两版同时挂载
```

## § 4 · 多轮序列架构

### 4.1 宏观规划

- **序列长度**:用户未指定 → 6-12 轮(skill 自决)
- **策略递进**:整套序列要有逻辑路径(痛点 → 价值 → 信任 → 差异化)

### 4.2 微观矩阵

- **每轮模板数量**:用户未指定 → 3-6 封(skill 自决)
- 每轮内模板**核心策略一致**,但在标题 / 开场 / 价值点强调 / 钩子之间变化

## § 5 · 最终交付格式(3 阶段)

### 阶段一 · 战略蓝图确认

格式:Markdown 引用块。

```markdown
> **战略蓝图确认**
> *   **核心洞察**: 目标客户(欧洲美妆采购经理)最大痛点是欧盟塑料禁令带来的合规风险
> *   **破局点**: 我们的 `EN13432 认证` 和 `30 天 90% 降解` USP 是直接解药
> *   **序列逻辑**: 以"合规解困"为尖刀,以"高端环保品牌价值"为核心,以"便捷合作"为助推
```

### 阶段二 · 营销序列总览

格式:Markdown 列表 · 写清 N 轮总数 + 每轮策略 + 目标。

### 阶段三 · 逐轮邮件矩阵

**分离输出**原则 · 带上下文信息的代码块。每轮每模板独立块:

````
**第 1 轮:策略 8(交期优势型)**

**【模板 1/4】**

```邮件主题 (第1轮 - 模板 1/4)
Kayaks in France next week? 🛶
```

```邮件正文 (第1轮 - 模板 1/4)
<p>Hi {联系人:名称},</p>   <!-- ⚠️ 占位示例,不是真变量 · 录入来发信后必用编辑器顶部【插入变量】按钮替换,见 references/09 关 2 · 这行注释生产时可删 -->
<p>...(HTML 正文,遵守呼吸感公式)...</p>
<p>Alex</p>
```

```中文对照 (第1轮 - 模板 1/4)
...
```
````

## § 6 · 执行铁律(最高优先级)

### 6.1 完整性铁律

- **绝对禁止**以任何理由(包括"为了简洁 / 模式重复")省略任何轮次或模板
- 8 轮 × 4 模板 = **必须**完整生成 32 个模板
- **严禁**输出末尾用 "...后续将遵循此结构..." 一类省略性描述

### 6.2 格式化铁律

- 每个独立 Markdown 代码块之间**必须**至少一个空行

### 6.3 中英对照铁律

- 输出**必须**附中文翻译(给操作者审稿用,不发给客户)
- 中文不是逐字翻,要写出**这封邮件的策略意图**让操作者明白

## § 7 · 用户指令模板

skill 让用户按这个格式提供:

```
1. 营销序列轮数: 8         (空则 skill 自决 6-12)
2. 每轮模板数量: 4         (空则 skill 自决 3-6)
3. 沟通风格: 专业直接       (可选:热情友好 / 高端专属 / 风趣创意)
4. 价值主张:
   - 产品/服务: ...
   - 特性 → 利益:
     · 特性 1 → 利益 1
     · 特性 2 → 利益 2
   - USP(尽量数字化): ...
5. 目标客户群体:
   - 行业 & 地区: ...
   - 目标职位: ...
   - 核心痛点: ...
6. 目标语言: 英语(默认地道欧美商业英语)
```

## § 8 · 常见踩坑

| 坑 | 后果 | 怎么避 |
|---|---|---|
| 写 "Dear Sir/Madam" | 一上来就群发感 | 用 `Hi {联系人:名称}`(⚠️ `{联系人:名称}` 是占位示例 · 录入来发信时**必用编辑器顶部【插入变量】按钮**替换 · 详见 `references/09-validation.md` 关 2) / `Hello` |
| 签名带公司+电话+网址 | 进垃圾邮件 | 只留英文名 |
| 第一封 200+ 词 | 阅读完成率掉 | 严守 ≤ 100 词 |
| 强调词 > 4 个 | 视觉污染 | 数量铁律 |
| 所有轮策略一致 | 客户审美疲劳 | 策略递进 |
| 直接给 30+ 封 | 模板趋同 / AI 省略 | 用对话状态机一轮一轮要 |
